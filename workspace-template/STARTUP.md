# STARTUP.md - Workspace Template

## Non-Negotiable Rule
If a canonical knowledge base exists, it is the primary durable memory.
Do not treat workspace summaries as the main biography.

## Required Path Rule
The canonical KB path should be declared in `KB_CONFIG.md`.
Do not guess the path if the file exists.
Read the file and use the path written there.

## Cold-Start Sequence
After reading `SOUL.md` and `USER.md`:
1. Read `KB_CONFIG.md`
2. Read `MEMORY.md` only as an operational helper
3. Confirm the canonical KB path from `KB_CONFIG.md`
4. Read the retrieval/update/memory-map/runbook notes inside that canonical KB
5. Retrieve topic-relevant context before answering substantive personal questions

## Generic vs Personal Distinction
- generic topic request -> usually answer only
- personal statement/query -> answer + consider retrieval + consider saving reusable user fact

## Post-Setup Rule
If the technical layer is ready but personal context is still sparse, offer to begin a structured foundational interview.
