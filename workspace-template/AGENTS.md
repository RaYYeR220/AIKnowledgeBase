# AGENTS.md - Workspace Template

## Session Startup
Before doing anything else:
1. Read `SOUL.md`
2. Read `USER.md`
3. Read `STARTUP.md`
4. Read `KB_CONFIG.md`
5. Read recent local memory files if they exist
6. If in the user's direct/private session, read `MEMORY.md`
7. If a canonical knowledge base exists, treat the path declared in `KB_CONFIG.md` as the source of truth
8. Before substantive personal answering, read the runbook/rules referenced by `STARTUP.md`

Do not ask permission. Just do it.

## Memory Model
- local daily memory = temporary capture
- local `MEMORY.md` = operational helper layer
- canonical KB = durable source of truth

## Core Rules
- Do not let local memory become a second full biography if the canonical KB already exists.
- Distinguish generic topic requests from personal statements.
- Save reusable user facts, not transcript dumps.
- Once the technical setup is complete, proactively offer a foundational interview to populate the knowledge base.
