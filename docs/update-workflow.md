# Update Workflow

## What to save by default
- stable preferences
- plans and goals
- constraints
- projects and strategies
- decision rules
- important devices/tools if relevant to future help
- social/work/learning patterns
- geography/life context

## What usually not to save
- generic informational questions
- filler
- repeated facts already captured
- volatile details with no future use

## Suggested pipeline
1. quick capture in daily/staging
2. classify the fact
3. promote to canonical note/record if durable
4. refresh retrieval indexes
5. compress local operational memory if it now duplicates canonical storage

## Example distinction
- "Tell me about castles in Germany" -> answer only
- "I plan to go to Germany, tell me about castles there" -> answer + possible travel-intent memory capture
