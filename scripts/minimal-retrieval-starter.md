# Minimal Retrieval Starter Path

If you want the simplest possible first version, start with this:

## Canonical store
- markdown notes in a single directory tree

## Retrieval v1
- keyword search only
- no semantic search yet
- manual or scripted reindex after note changes

## Retrieval v2
Add semantic search later if needed:
- local embeddings
- remote embeddings
- managed vector DB
- hybrid keyword + semantic reranking

## Why this starter path matters
Many people do not need a full vector setup on day one.
A clean canonical KB + startup layer + keyword search is already enough to validate behavior.
