# Retrieval Workflow

## Step 1 — Inspect the current request
Classify the message:
- generic informational
- personal informational
- planning/decision request
- memory update / preference statement
- project status / ongoing thread

## Step 2 — Decide whether personal context matters
Retrieve if context could improve:
- precision
- realism
- tone fit
- prioritization
- constraints
- recommendations

## Step 3 — Retrieve from canonical storage first
Use hybrid retrieval when possible:
- keyword search
- semantic search
- merge/rerank

## Step 4 — Use operational memory only as support
Only for:
- recent captures
- startup orientation
- pointer summaries

## Step 5 — Answer with the smallest sufficient context
Do not flood the answer with unrelated biography.

## Step 6 — If meaningful new information was learned
- capture quickly in staging/daily layer
- promote durable facts to canonical storage
- refresh indexes
